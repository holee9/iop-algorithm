# SPEC-VGRID-001 — T8 WP9 커널 Virtual grid (SKS scatter 보정) 처리 모듈 (modules/)

XDET 영상처리 SW P1의 아홉 번째 작업 T8. **물리 anti-scatter grid가 없는 검출기**에서 산란(scatter) 성분을 처리로 추정·감산하여 grid의 산란 억제 효과를 "가상"으로 재현하는 작업 **WP9(SWR-1101~1103, FR-M008)**를 전용 처리 모듈 `modules/virtual_grid.py`로 T0 프레임워크의 단일 계약 `process(XFrame, CalibSet, Params) -> XFrame` 순수함수형으로 구현한다. 핵심 알고리즘은 **SKS(Scatter Kernel Superposition)** [L — 공개 학술 방법론: pencil-beam scatter PSF, 두께 의존 커널, 반복 보정]다 — 다운샘플(×8) 도메인에서 두께 프록시·kV 파라미터로 형상화한 이중 Gaussian 커널 K로 S = conv(P̂, K)를 반복(P̂₀ = I_down, P̂_{i+1} = I_down − S_i, 2~3회)하여 산란 S를 추정하고(SWR-1101), bilinear 업샘플 후 I′ = I − w·S↑(w = SW grid ratio 환산 계수, 사용자 선택 3:1~12:1 상당)로 감산하되 **비음수 제약 + 저신호 영역 w 자동 감쇠(노이즈 부스트 방지)**를 적용한다(SWR-1102). 이 스테이지는 고정 파이프라인 순서에서 전용 `virtual_grid` 스테이지(물리 grid line 억제 `grid`와 잡음 억제 `denoise` 사이)에서만 실행된다(결정 1, 「결정 필요/확인 사항」 1). 산란 커널 K는 **패널 실측 피팅([B], SWR-1101)**으로 얻는 검출기 캘리브레이션이므로(SWR-000-10이 "scatter 커널"을 CalibSet 종류로 명시) 신규 `CalibKind.SCATTER` + 오프라인 빌더 `metrics/scatter_kernel.py`가 방출하는 `CalibSet(SCATTER)`를 단일 소스로 소비한다(결정 2, DENOISE NOISE-kind 선례). **⚠P — SKS 수식을 청구하는 유효 특허(US 11,911,202 등)가 확인**되었으므로(SWR-1101/1103) 구현하되 릴리스 전 특허 청구항 대조 플래그를 유지하고(CLAUDE.md T8 "구현하되 릴리스 전 특허 대조 플래그 유지"), 회피 대안(MC 사전계산 LUT 커널, SWR-1103)을 예비 정의로 예약한다. **실측 scatter 취득 세트(GDS-scatter 팬텀) 도착 전에는 기지 산란(기지 커널로 주입한 저주파 veiling)을 담은 합성 팬텀으로 CNR 개선·산란 추정 정확도·노이즈 부스트 억제를 선검증**한다.

- 근거: SWR-1101~1103(커널 Virtual grid, FR-M008, ⚠P) · SWR-000-1~12(아키텍처, 특히 SWR-000-9 공용 컴포넌트 ① pyramid, SWR-000-10 CalibSet "scatter 커널" 종류) — `docs/XDET_SWR_spec_v1.2.md`; EVAL v1.1 XDET-EV-202(Scatter correction: CNR 개선 ≥ +20% min / 무그리드+보정 vs 물리 grid 비열등[관찰자] / DL scatter 추정 편차[MC 기준, DL·Gen2]); TestSpec XDET-TC-017(무그리드+커널 보정 CNR 개선, GDS-scatter 팬텀, EV-202 min); FR-M008

T0 표면(`CANONICAL_ORDER`) 변경 run-blocking. 기각 대안: `denoise` 뒤(산란은 저주파라 denoise가 추정을 돕지 못하고, 감산 노이즈 부스트가 정리되지 않음).
  2. **신규 `CalibKind.SCATTER` + 종류-단계 배선([run-blocking])**: 산란 커널 K(이중 Gaussian 형상, 두께 프록시·kV 의존)는 **패널 실측 피팅([B], SWR-1101)**으로 얻는 진짜 검출기 캘리브레이션이며 SWR-000-10이 "scatter 커널"을 CalibSet 종류로 명시하므로, 신규 `CalibKind.SCATTER` + `_KIND_BY_STAGE["virtual_grid"] = "scatter"` + `SCATTER_PAYLOAD_KEYS`(이중 Gaussian 계수·두께/kV 의존, 스키마 [B] 이연)를 신설한다. 오프라인 빌더 `metrics/scatter_kernel.py`(lag_irf/noise_model/defect_map 선례, `metrics → common`)가 `CalibSet(SCATTER)`를 방출하고 `virtual_grid` 모듈은 이를 단일 소스로 소비하며 XFrame 기본값·부재 시 침묵 기본값 없이 REJECT한다(DENOISE NOISE-kind 결정 2 선례 — OTHER 재사용 대신 SCATTER 신설로 종류-단계 게이트 결속을 강제해 침묵 기본값 대체를 구조적으로 차단). **T7 grid와의 구별**: grid는 검출기 캘리브레이션이 없어 CalibSet(OTHER) 빈 placeholder를 소비했으나(POST-001 선례), virtual_grid는 실측 산란 커널이라는 진짜 캘리브레이션을 가진다. T0 표면(`common/calibset.py` + orchestrator `_KIND_BY_STAGE`) 변경 run-blocking.
  3. **비음수 제약 + 저신호 w 자동 감쇠(SWR-1102)**: 감산 I′ = I − w·S↑에서 grid ratio 환산 계수 w(사용자 선택 3:1~12:1 상당)는 캘리브레이션이 아닌 **사용자 선택 Params**([T]/[P])로 외부화하고, 저신호 영역에서 w를 자동 감쇠해 노이즈 부스트를 억제하며 출력 화소의 비음수를 보장한다(REQ-VGRID-SUBTRACT-2/3).
  4. **공용 컴포넌트 ① pyramid 소비(SWR-000-9) — 공간 도메인**: SKS 다운샘플(×8)은 `common/pyramid.py`의 Gaussian `reduce_once`(×2 3회)를 소비해 중복 다운샘플 구현을 피하고(SWR-000-9 ① pyramid 소비자), 업샘플은 SWR-1102가 명시한 bilinear(scipy, pyramid의 Gaussian expand와 구별)를 쓴다. **T7 grid(주파수 도메인 → `common/fft_psd.py`)와 달리 virtual_grid는 공간 도메인(SKS 컨볼루션·다운/업샘플)이므로 `common/fft_psd.py`가 아니라 `common/pyramid.py`를 소비**한다. 이중 Gaussian 커널 컨볼루션은 자체 scipy 공간 필터로 구현한다.

  - 파라미터 등급 확정(SWR 부록 A/A-2 대조): 산란 커널 K(이중 Gaussian 계수·두께/kV 의존, SWR-1101)=**TBD-[B]**(부록 A: SWR-1101 패널 실측); SKS 방법론(pencil-beam PSF·반복 구조)=**[L]**(부록 A-2: 1101 SKS); 반복 횟수 2~3(SWR-1101)=SWR 본문 명시([L] 근거, 반복 횟수 자체 무등급 [T] 등재 요청); grid ratio 환산 계수 w(SWR-1102)=사용자 선택 [T]/[P](무등급 등재 요청); 저신호 감쇠 임계·감쇠 곡선(SWR-1102 "저신호 영역 w 자동 감쇠")=미정량(무등급 [T] 등재 요청); CNR 개선 허용값(EV-202 min)·산란 추정 정확도 허용오차·노이즈 부스트 허용오차=EV/[T] 외부 주입. 다운샘플 배율 ×8·업샘플 bilinear(SWR-1101/1102)=SWR 본문 명시(방법 [L], 배율 무등급 등재 요청).

## Environment / Assumptions

- Python 3.11+, numpy/scipy 기반 float 골든 모델(tech.md). **속도 최적화 금지 — 정확도 단일 목표**(P2에서 최적화). 현재 `pyproject.toml` 의존성은 numpy/scipy 전용이며, 다운샘플(`common/pyramid.py` 소비)·이중 Gaussian 컨볼루션·bilinear 업샘플·감산을 전부 numpy/scipy로 구현하므로 신규 의존성을 추가하지 않는다.
- 대상 프레임: 3072×3072 / 3072×2560, 16-bit unsigned raw, pitch 140µm(EVAL v1.1 §0). 산란은 물체 두께 분포에 의존하는 **장면 종속 저주파 veiling glare**이므로 평판 플랫필드 캘리브레이션으로 제거할 수 없고 영상별로 SKS로 추정·감산해야 한다(오프셋/게인 보정과 구별).
- **⚠P 특허 플래그(SWR-1101/1103, CLAUDE.md T8) — 명시적 유지**: SKS 수식을 청구하는 **유효 특허(US 11,911,202 등)가 확인**되었다. 방법 자체(SKS)는 공지 학술 기술이나 구현 세부(커널 형상·반복 구조)의 청구항 대조는 **릴리스 게이트로 이연**(사용자 지시, SWR 본문 "특허 검토는 릴리스 게이트로 이연 … ⚠P 표기"·§8/11/5 ⚠P 절 공통 정책)한다. **P1 SW 구현은 특허 판단을 내리지 않고 ⚠P 플래그를 유지한 채 기술 필요성 기준으로 진행**하며, 특허 클리어런스가 릴리스 게이팅 조건이다(SW 구현 범위 밖, Exclusions 참조). 회피 대안(MC 사전계산 LUT 커널, SWR-1103)은 예비 정의로 예약하되 P1에 구현하지 않는다.
- **실측 scatter 취득 세트 도착 전 — 합성 팬텀으로 모듈을 검증한다.** 기지 산란 커널로 기지 저주파 산란을 주입한 합성 GDS-scatter 팬텀(기지 대비 신호·기지 배경)으로 CNR 개선·산란 추정 정확도·노이즈 부스트 억제를 확인한다. 산란 커널 파라미터는 [B](2-7 실측 피팅) 이연이며 합성 커널로 선검증한다(LAG 합성 IRF·DENOISE 합성 λ 선례).
- **용어 정의 — 합성 검증 컨텍스트(synthetic-validation context)**: 합성 팬텀 fixture 시험 실행(기지 산란 커널·대비 신호 주입 → 보정 후 판정)을 가리키는 단일 용어로, T0(SPEC-INFRA-001)의 검증 모드(validation_mode)와는 별개 개념이다. SPEC-CORR/LNSG/LAG/DENOISE/POST/GRID의 동명 정의를 계승한다.
- T0 계약 소비: `virtual_grid` 모듈은 XFrame(불변)을 입력받아 새 XFrame을 반환하는 **무상태 처리 모듈**이며(lag과 달리 내부 상태 없음) `process(XFrame, CalibSet, Params) -> XFrame` 계약을 따른다. 마스크 스택 비트플래그·이력 체인·오케스트레이터 진입 게이트·`CANONICAL_ORDER`를 그대로 소비한다. 의존 방향은 `modules → common` 단방향(import-linter). 지표(CNR·산란 정확도 대조) 판정은 `tests/`에서만 소비한다(모듈은 `metrics` 미import).
- **공용 컴포넌트 ① pyramid 소비(SWR-000-9) — 공간 도메인**: SKS 다운샘플(×8)은 `common/pyramid.py`의 Gaussian `reduce_once`(×2 3회, T6 구현)를 소비해 중복 다운샘플을 피한다. 업샘플은 SWR-1102가 명시한 bilinear(scipy, pyramid의 Gaussian `expand`와 구별). **T7 grid와의 계층 구별**: grid는 주파수 도메인 notch로 `common/fft_psd.py`(③ FFT·PSD)를 소비했으나, virtual_grid는 공간 도메인 SKS 컨볼루션·다운/업샘플이므로 `common/fft_psd.py`가 아니라 `common/pyramid.py`(① pyramid)를 소비한다. FFT/PSD·pyramid를 모듈 내부에 재구현하지 않는다.
- **산란 커널 = 검출기 캘리브레이션(SWR-000-10/1101 [B])**: 커널 K는 신규 `CalibSet(SCATTER)`(패널 ID·해상도·유효기간·이중 Gaussian payload·두께/kV 의존)로 소비하며, 오프라인 빌더 `metrics/scatter_kernel.py`가 방출한다. 모듈은 이를 **단일 소스**로 소비하고 부재·퇴화 시 침묵 기본값 없이 REJECT한다(DENOISE NOISE·LAG IRF·raw_saturation_threshold 무-침묵-기본값 선례). CalibSet payload 스키마(이중 Gaussian 계수·두께군별 커널 전환 등)는 [B] 실측 도착 시 확정한다.
- **T7 grid와의 기능 구별(핵심)**: `grid`(T7, SWR-1001~1006)는 **영상에 이미 존재하는 물리 anti-scatter grid의 격자선(주기 변조, aliased 가능)을 주파수 notch로 제거**한다(물리 grid 장착 검출기 대상, 산란 추정·감산 없음). `virtual_grid`(T8, SWR-1101~1103)는 **물리 grid가 없는 검출기에서 산란을 SKS로 추정·감산해 grid의 산란 억제 효과를 가상으로 재현**한다(무그리드 검출기 대상, 격자선 제거 없음 — 애초에 격자선이 없음). 두 스테이지는 취득 컨텍스트상 상호배타이며 상대 순서 `grid → virtual_grid`는 저비율 물리 grid의 잔존 산란이라는 가정 경계 사례를 위한 것이다.
- 물리·문헌·튜닝·무등급 상수(산란 커널 K [B], 반복 횟수 2~3, 다운샘플 ×8, grid ratio 환산 계수 w, 저신호 감쇠 임계·곡선)는 전부 **Params/CalibSet 단일 소재**로 외부화한다(하드코딩 금지). K는 `CalibSet(SCATTER)`, w·반복 횟수·저신호 감쇠 파라미터는 Params. 등급은 SWR 부록 A/A-2를 따르며 미등재 항목은 등재 요청한다(TBD-[B]: 커널 K; TBD-[T]/무등급 등재 요청: w·저신호 감쇠·반복 횟수·다운샘플 배율).
- EV 판정 수치(EVAL v1.1 EV-202 CNR 개선 ≥ +20% min)·산란 추정 정확도 허용오차·노이즈 부스트 허용오차는 **엔진·모듈 외부에서 주입**된다(측정=판정 분리, METRICS/CORR/LNSG/LAG/DENOISE/POST/GRID 계승). 처리 모듈·판정 코드는 게이트 임계를 내장하지 않는다.

## Requirements (EARS)

### REQ-VGRID-ESTIMATE — SKS 산란 추정(다운샘플 반복 커널 중첩) (SWR-1101, FR-M008)

- **REQ-VGRID-ESTIMATE-1 (Event-Driven)** — WHEN `virtual_grid` 모듈이 입력 XFrame을 처리하면, THEN 시스템은 입력 영상을 다운샘플(×8)한 도메인에서 SKS 반복 — P̂₀ = I_down, S_i = conv(P̂_i, K), P̂_{i+1} = I_down − S_i — 을 Params가 지정한 횟수(2~3회, [L] 근거)만큼 수행하여 산란 추정 Ŝ를 산출해야 한다(SWR-1101). 다운샘플은 공용 컴포넌트 `common/pyramid.py`의 Gaussian `reduce_once`(×2 3회)를 소비하며 다운샘플을 모듈 내부에 재구현하지 않는다(SWR-000-9 ①). 반복 횟수·다운샘플 배율은 Params로 외부화한다(하드코딩 금지).
- **REQ-VGRID-ESTIMATE-2 (Event-Driven)** — WHEN SKS 커널 컨볼루션을 수행하면, THEN 시스템은 두께 프록시·kV 파라미터로 형상화한 **이중 Gaussian 커널 K**를 `CalibSet(SCATTER)` payload에서 취해 사용해야 한다(SWR-1101; 커널 K = TBD-[B] 패널 실측). 커널 계수는 CalibSet(SCATTER) 단일 소스에서만 도출되며 모듈은 커널을 내부 하드코딩하지 않는다(REQ-VGRID-CALIB-1).
- **REQ-VGRID-ESTIMATE-3 (Unwanted)** — IF `virtual_grid` 모듈이 점근·근사 없이 진행돼야 할 산란 추정을 커널 부재로 임의 기본 커널이나 대역통과 근사로 대체하려 하면, THEN 시스템은 이를 계약 위반으로 취급해야 한다. 산란 추정은 `CalibSet(SCATTER)` 커널을 소비하는 결정론적 단일 경로이며 침묵 기본 커널 분기가 없다(무-침묵-기본값, REQ-VGRID-CALIB-2와 연동).

### REQ-VGRID-SUBTRACT — 산란 감산(업샘플·grid ratio 가중·비음수·저신호 감쇠) (SWR-1102, FR-M008)

- **REQ-VGRID-SUBTRACT-1 (Event-Driven)** — WHEN 산란 추정 Ŝ가 산출되면, THEN 시스템은 Ŝ를 bilinear로 원해상도 업샘플(Ŝ↑)한 뒤 I′ = I − w·Ŝ↑로 감산해야 한다(SWR-1102). w는 SW grid ratio 환산 계수(사용자 선택 3:1~12:1 상당)로 **Params**에서 취한다([T]/[P], 캘리브레이션 아님). 업샘플은 SWR-1102가 명시한 bilinear이며(pyramid Gaussian expand와 구별) w는 Params로 외부화한다(하드코딩 금지).
- **REQ-VGRID-SUBTRACT-2 (Event-Driven)** — WHEN 저신호 영역에서 감산이 노이즈를 부스트할 수 있으면, THEN 시스템은 해당 영역의 유효 감산 가중 w를 자동 감쇠하여 노이즈 부스트를 억제해야 한다(SWR-1102 "저신호 영역 w 자동 감쇠"). 저신호 감쇠 임계·감쇠 곡선은 Params로 외부화한다(하드코딩 금지). 트리거 산란은 REQ-VGRID-ESTIMATE가 산출한다(ESTIMATE → SUBTRACT 상류 추적).
- **REQ-VGRID-SUBTRACT-3 (Unwanted)** — IF 감산 결과가 화소를 음수(물리적으로 불가능한 음의 X선 신호)로 만들면, THEN 시스템은 비음수 제약을 적용해 출력 화소가 음수가 되지 않도록 해야 한다(SWR-1102 "비음수 제약"). 결정론적 단일 경로 — 비음수 보장 분기 외 임의 재스케일이 없다.

### REQ-VGRID-CALIB — 산란 커널 캘리브레이션 소스 + ⚠P 회피 대안 예약 (SWR-000-10, SWR-1101/1103 [B]·⚠P)

- **REQ-VGRID-CALIB-1 (Ubiquitous)** — `virtual_grid` 모듈은 산란 커널 K를 신규 `CalibSet(SCATTER)`(패널 ID·해상도·유효기간·이중 Gaussian payload·두께/kV 의존)에서 **단일 소스**로 소비해야 하며(SWR-000-10 "scatter 커널" 종류), 오프라인 빌더 `metrics/scatter_kernel.py`(`metrics → common` 단방향, lag_irf/noise_model/defect_map 선례)가 이를 방출한다. 모듈은 커널을 내부 하드코딩하지 않는다. CalibSet payload 스키마(이중 Gaussian 계수·두께군별 커널 전환 등)는 [B] 실측 도착 시 확정한다.
- **REQ-VGRID-CALIB-2 (Unwanted)** — IF `CalibSet(SCATTER)`가 부재하거나 payload가 퇴화(빈 커널·비유한 계수)이면, THEN 시스템은 침묵 기본값 대체 없이 명시 오류로 REJECT해야 한다(SWR-000-5 캘리브레이션 부재 기본값 무단 대체 금지, DENOISE NOISE·LAG IRF·raw_saturation_threshold 선례). 결정론적 단일 경로 — 기본 커널 대체 분기가 없다.
- **REQ-VGRID-CALIB-3 (Ubiquitous)** — 산란 커널을 `CalibSet(SCATTER)`에서 전량 소싱하는 설계로, ⚠P 회피 대안(MC 사전계산 LUT 커널, SWR-1103)을 모듈 재설계 없이 **빌더 단(`metrics/scatter_kernel.py`)에서 대체 커널 도출로 치환 가능**하도록 예비 정의로 예약해야 한다. 회피 대안의 실제 구현은 P1 범위 밖이다(릴리스 게이트 대비, Exclusions 참조). 특허 청구항 대조·릴리스 게이팅은 SW 구현이 판단하지 않으며 ⚠P 플래그를 유지한다(「결정 필요/확인 사항」 5).

### REQ-VGRID-CONTRACT — 공통 모듈 계약 준수 (SWR-000-2~12, REQ-INFRA-* 소비)

- **REQ-VGRID-CONTRACT-1 (Ubiquitous)** — `virtual_grid` 모듈은 단일 시그니처 `process(XFrame, CalibSet, Params) -> XFrame` 순수함수형이어야 하며(SWR-000-7, REQ-INFRA-CONTRACT-1), 입력 XFrame을 불변으로 취급(원본 미변경)하고 새 XFrame을 반환해야 한다(SWR-000-3, REQ-INFRA-DATA-6). 모듈은 내부 상태를 보유하지 않는다(무상태).
- **REQ-VGRID-CONTRACT-2 (Event-Driven)** — WHEN `virtual_grid` 모듈이 출력 XFrame을 생성하면, THEN 시스템은 처리 메타(모듈 버전 · 파라미터 해시 · 소비 CalibSet(SCATTER) ID)와 스칼라 진단(반복 횟수 · 적용 grid ratio w · 추정 산란 비율(scatter fraction) · 저신호 감쇠 적용 여부 · 비음수 클램프 화소 수 · ⚠P 커널 provenance 플래그 등)을 이력 체인 엔트리(`HistoryEntry.extra`)에 결정론적으로 추가해야 한다(SWR-000-4, REQ-INFRA-DATA-4, IEC 62304 추적).
- **REQ-VGRID-CONTRACT-3 (Ubiquitous)** — 의존 방향은 `modules → common` 단방향이어야 하며, `virtual_grid` 모듈은 다른 처리 모듈 · `pipeline` · `metrics`를 import해서는 안 된다(SWR-000-8, REQ-INFRA-STATIC import-linter 계약). 실행 순서·조합은 오케스트레이터 단독 소관이며 모듈 간 직접 호출은 금지된다(REQ-INFRA-ORCH-1/2). 다운샘플은 공용 컴포넌트 `common/pyramid.py`(① pyramid)를 소비·재사용하며 다운샘플을 모듈 내부에 재구현하지 않는다(SWR-000-9). CNR·산란 정확도 판정은 `tests/`에서만 이뤄진다(모듈은 `metrics` 미import).
- **REQ-VGRID-CONTRACT-4 (Unwanted)** — IF `virtual_grid` 모듈이 XFrame 컨테이너 외 채널(전역 상태 · 부가 반환값 · 파일 우회)로 데이터를 전달하려 하면, THEN 시스템은 이를 계약 위반으로 취급해야 한다(SWR-000-6 사이드채널 금지). 자동 검출 가능 범위는 시그니처·부가 반환값 위반(계약 검사)과 의존 방향 위반(import-linter 정적 검사)이며(acceptance.md EC의 검증 범위와 동일), 전역 상태·파일 우회는 테스트 가능 AC가 아닌 설계 규칙으로서 코드 리뷰 게이트로 다룬다(SPEC-INFRA-001 REQ-INFRA-DATA-2 방식 계승).
- **REQ-VGRID-CONTRACT-5 (Ubiquitous)** — `virtual_grid` 모듈은 고정 파이프라인 순서 `CANONICAL_ORDER`의 전용 `virtual_grid` 스테이지(`grid`와 `denoise` 사이, 결정 1)에서만 실행되어야 하며(SWR-000-2, REQ-INFRA-ORCH-3; 등록 stages는 `CANONICAL_ORDER`의 부분수열), 합성 입력 + 기대 출력 fixture로 harness 단독 시험이 가능해야 한다(SWR-000-11, XDET-TC-000). 스테이지는 실측 산란 커널이라는 검출기 캘리브레이션을 가지므로 `_KIND_BY_STAGE["virtual_grid"] = "scatter"`로 종류-단계를 배선하고 진입 게이트에서 `CalibSet(SCATTER)`(종류·패널 ID·해상도·유효기간 일치)를 소비한다(결정 2; T7 grid의 CalibSet(OTHER) 무-캘리브레이션과 구별).
- **REQ-VGRID-CONTRACT-6 (Unwanted)** — IF `virtual_grid` 모듈이 포화 화소 값을 "복원"하거나(SWR-602 [HARD] 복원 금지 계승) 마스크 플래그를 신규 설정·해제하면, THEN 시스템은 이를 계약 위반으로 취급해야 한다. 모듈은 마스크를 소비만 하고 substrate를 변경하지 않으며 포화 영역을 재구성하지 않는다(포화 화소 감산 시에도 클리핑으로 소실된 정보를 복원하지 않음). 결정론적 단일 경로.

### REQ-VGRID-VALIDATE — 합성 검증(CNR 개선 + 산란 추정 정확도 + 노이즈 부스트 억제) (XDET-TC-017, EV-202, FR-M008)

- **REQ-VGRID-VALIDATE-1 (State-Driven)** — WHILE 실측 scatter 세트 도착 전 합성 검증 컨텍스트인 동안, 시스템은 기지 산란 커널로 기지 저주파 산란을 주입한 합성 GDS-scatter 팬텀(기지 대비 신호·기지 배경)으로 CNR 개선·산란 추정 정확도·노이즈 부스트 억제를 보여야 한다(DoD 전제, CLAUDE.md T8 "합성 데이터로 선검증", LAG 합성 IRF·DENOISE 합성 λ 선례).
- **REQ-VGRID-VALIDATE-2 (Event-Driven)** — WHEN 합성 GDS-scatter 팬텀에 virtual grid 보정을 적용하면, THEN 시스템은 `tests/`에서 보정 후 관심/배경 ROI의 CNR 개선율이 EV-202 min(≥ +20%) 이상임을 결정론적으로 이진 판정 가능해야 한다(XDET-TC-017 "무그리드+커널 보정 CNR 개선", EV-202 min). CNR은 `tests/`에서 기지 ROI 통계(`common/robust_stats` 소비)로 산출하고 EV-202 임계는 외부 주입한다(측정=판정 분리). 이것이 T8의 하드 DoD이다.
- **REQ-VGRID-VALIDATE-3 (Event-Driven)** — WHEN 기지 커널로 기지 산란 S_inj를 주입한 합성 팬텀에 적용하면, THEN 시스템은 `tests/`에서 추정 산란 Ŝ가 주입 산란 S_inj와 허용오차([T] 외부 주입) 이내로 일치함을 판정 가능해야 한다(SKS 추정 정확도; EV-202 "DL scatter 추정 편차[MC 기준]"는 DL/Gen2 이연이나 SKS 추정 정확도 원리를 기지-주입 대조로 P1에 적용). 허용오차는 외부 주입한다(측정=판정 분리).
- **REQ-VGRID-VALIDATE-4 (Event-Driven)** — WHEN 저신호 영역을 포함한 합성 팬텀에 적용하면, THEN 시스템은 `tests/`에서 (a) 저신호 영역에서 보정 후 노이즈가 허용오차([T]) 초과로 증가하지 않고(w 자동 감쇠, SWR-1102 노이즈 부스트 방지), (b) 출력 화소가 음수가 되지 않음(비음수 제약)을 판정 가능해야 한다(REQ-VGRID-SUBTRACT-2/3 검증). 노이즈 부스트 허용오차는 외부 주입한다.
- **REQ-VGRID-VALIDATE-5 (Ubiquitous)** — EV-202(CNR 개선 min)·산란 추정 정확도·노이즈 부스트 판정 수치는 EVAL v1.1/Params에서 외부 주입되어야 하며, 검증은 산출값과 외부 임계의 비교로만 이뤄져야 한다(측정=판정 분리 계승). 처리 모듈·판정 코드는 게이트 임계를 내장하지 않는다. 관찰자 비열등(EV-202 "무그리드+보정 vs 물리 grid 관찰자")은 인허가 이연(PARTIAL)이며 개발 게이트가 아니다. 시험 케이스 XDET-TC-017은 현재 pytest skeleton(skip)에서 합성 입력·판정 연동의 실동작 케이스로 전환되어야 한다(REQ-INFRA-CI-1 계승; 모듈은 `metrics` 미import이므로 CNR·정확도 판정은 `tests/`에서 수행).

## Exclusions (What NOT to Build)

- **격자선(grid line) 제거 없음** — virtual_grid는 물리 grid가 없는 검출기의 산란을 보정할 뿐 격자선을 제거하지 않는다. 물리 grid line 억제(SWR-1001~1006/T7, `modules/grid.py`)는 별개 스테이지(`grid`)로 이미 처리된다. 두 스테이지는 취득 컨텍스트상 상호배타이며 virtual_grid는 grid의 주파수 notch를 재수행하지 않는다.
- **특허 클리어런스·릴리스 게이팅 없음** — [⚠P] SKS 수식 특허(US 11,911,202 등) 청구항 대조·특허 판단·릴리스 게이팅은 **P1 SW 구현 범위 밖**이며 법무/특허 대조 릴리스 게이트 소관이다. SW 구현은 ⚠P 플래그를 유지한 채 기술 필요성 기준으로 진행하고 특허 판단을 내리지 않는다(SWR-1101/1103, CLAUDE.md T8 "구현하되 릴리스 전 특허 대조 플래그 유지"). 「결정 필요/확인 사항」 5 참조.
- **MC 사전계산 LUT 커널 구현 없음** — 회피 대안(SWR-1103 MC 사전계산 LUT 커널)은 커널을 `CalibSet(SCATTER)`에서 소싱하는 설계로 빌더 단 치환이 가능하도록 예비 정의로만 예약하고 P1에 구현하지 않는다(릴리스 게이트 특허 대조 결과에 따른 우발 대안).
- **DL scatter 추정 없음** — EV-202의 "DL scatter 추정 편차(MC 기준)" 행은 DL 경로로 Gen 2 예약 항목이며 P1 범위 밖이다(CLAUDE.md "Gen 2 항목(DL, ADR)은 구현하지 않는다"). P1은 SKS(공지 학술 방법)만 구현한다.
- **두께군별 커널 전환(ASKS류) 없음** — SWR-1101 "두께군별 커널 전환(ASKS류) 승급 경로 예약"은 승급 경로로만 예약하고 P1은 단일 두께-프록시 이중 Gaussian 커널을 구현한다. 커널 전환은 [B] 실측·후속 범위.
- **산란 커널 실측 피팅 파이프라인 게이트 없음** — 산란 커널 K는 [B](2-7 실측 피팅) 이연이며 P1은 합성 커널로 선검증한다. 실측 도착 후 커널 치환은 게이트가 아니다(부록 A 정책). 커널 피팅 빌더 `metrics/scatter_kernel.py`는 오프라인 캘리브레이션 도구로 파이프라인 스테이지가 아니다.
- **후속·타 WP 처리 모듈 없음** — NDT(SWR-1201~1204/T9)·티어·동일성(SWR-1301~1303/T10)은 T8 범위 밖. 선행 offset/gain/defect(T2)·lag(T4)·line noise/포화/기하(T3)·grid line 억제(T7)·denoise(T5)·MSE/DRC·윈도잉/GSDF(T6)도 본 SPEC 범위 밖(이미 구현됨).
- **`post` 스테이지 실현 없음** — 결정 1로 전용 `virtual_grid` 스테이지를 `grid`와 `denoise` 사이에 신설하며 `post`는 예약 tail로 유지한다(T5·T6·T7 "기존 post 실현" 기각 계승).
- **관찰자 비열등 판정 없음** — EV-202 "무그리드+보정 vs 물리 grid(관찰자) 비열등"의 관찰자 판독은 인허가 제출용 관찰자 연구 대상으로 개발 게이트가 아니다(PARTIAL). T8은 CNR 개선(결정론)·산란 추정 정확도·노이즈 부스트 억제로만 판정한다.
- **신규 마스크 플래그 없음** — 산란 비율·경고·진단은 `HistoryEntry.extra`로 다루며 XFrame 마스크 스택에 신규 플래그를 도입하지 않는다(T0 표면 무변경).
- **의존성 추가 없음** — 다운샘플(`common/pyramid.py` 소비)·이중 Gaussian 컨볼루션·bilinear 업샘플·감산 전부 numpy/scipy 자체 구현으로 `pyproject.toml` 신규 의존성 미추가.
- **EV 게이트 임계·허용오차 내장 없음** — EV-202 CNR min·산란 추정 정확도·노이즈 부스트 허용오차는 외부 주입. 처리 모듈·판정 코드는 합격/불합격 임계를 내장하지 않는다.
- **성능·처리시간·티어 게이트 없음** — EV-401/402, XDET-TC-020/021은 P2.

## 결정 필요/확인 사항

SWR 조항이 T0/T1 구현과 모호하거나 상충하는 지점. **「1」·「2」는 run-blocking 확정 대상**(T0 표면 변경 — `CANONICAL_ORDER` 스테이지 신설 및 `CalibKind.SCATTER`/`_KIND_BY_STAGE` 신설). **「3~5」는 확인 대상**으로 각 항목에 가정 default를 명시한다. 최종 확정은 orchestrator 결정(plan-audit)을 따른다.

1. **[run-blocking] virtual_grid 파이프라인 스테이지 배치** — 전용 `virtual_grid` 스테이지를 `grid`와 `denoise` **사이**에 신설한다(`CANONICAL_ORDER = … → geometry → grid → virtual_grid → denoise → mse → window → post`). `post`는 예약 tail로 유지. **rationale**: (a) 기하 보정된 축-정렬 영상에서 산란을 추정하므로 `geometry` 뒤; (b) 산란은 저주파 신호 보정이고 감산의 저신호 노이즈 부스트를 후속 BM3D(`denoise`)가 정리하므로 `denoise` 앞; (c) 산란은 대비(CNR)를 저하시키므로 MSE/DRC 대비강화(`mse`, WP6)가 잔존 산란 구배를 증폭하기 전에 제거해야 함. **T7 grid와의 관계(사용자 질의 명시)**: `grid`(물리 grid line 억제)와 `virtual_grid`(무그리드 검출기 산란 보정)는 취득 컨텍스트상 **상호배타**(검출기가 물리 grid를 장착하면 T7, 미장착이면 T8)이며, 저비율 물리 grid의 잔존 산란 처리라는 가정 경계 사례를 위해 `grid → virtual_grid` 상대 순서를 둔다. 등록 stages가 `CANONICAL_ORDER` 부분수열이면 통과하므로 스테이지 신설은 하류 미등록 SPEC에 하위호환(T5·T6·T7 선례). **기각 대안**: `denoise` 뒤(`… → denoise → virtual_grid → mse …`) — 산란은 저주파라 denoise가 추정을 돕지 못하고, 감산 노이즈 부스트가 후속 정리 없이 남는다. T0 표면(`CANONICAL_ORDER`) 변경 run-blocking — plan-audit 확정 필요.
2. **[run-blocking] 신규 `CalibKind.SCATTER` + 종류-단계 배선** — 산란 커널 K는 패널 실측 피팅([B], SWR-1101)으로 얻는 진짜 검출기 캘리브레이션이며 SWR-000-10이 "scatter 커널"을 CalibSet 종류로 명시한다. **가정 default**: 신규 `CalibKind.SCATTER` + `_KIND_BY_STAGE["virtual_grid"] = "scatter"` + `SCATTER_PAYLOAD_KEYS`(이중 Gaussian 계수·두께/kV 의존, 스키마 [B] 이연)를 신설하고, 오프라인 빌더 `metrics/scatter_kernel.py`가 `CalibSet(SCATTER)`를 방출하며 모듈은 단일 소스로 소비·부재 시 REJECT한다(DENOISE NOISE-kind 결정 2 선례 — OTHER 재사용 대신 SCATTER 신설로 종류-단계 게이트 결속을 강제해 침묵 기본값 대체를 구조적으로 차단). T7 grid는 무-캘리브레이션 CalibSet(OTHER)였으나 virtual_grid는 실측 커널이라는 진짜 캘리브레이션을 가진다. T0 표면(`common/calibset.py` + orchestrator `_KIND_BY_STAGE`) 변경 run-blocking. 확인: SCATTER payload 정확 스키마(이중 Gaussian 계수 수·두께/kV 축), 빌더의 실측-피팅 vs 합성-커널 선검증 경로.
3. **[확인] grid ratio 환산 계수 w 소재 + 비음수·저신호 감쇠 파라미터** — SWR-1102 w(SW grid ratio 환산 계수, 사용자 선택 3:1~12:1 상당) 소재와 저신호 감쇠·비음수 처리 상세. **가정 default**: w는 캘리브레이션이 아닌 **사용자 선택 Params**([T]/[P])로 외부화(CalibSet 미탑재). 저신호 감쇠 임계·감쇠 곡선, 비음수 제약 처리(클램프 vs 제약 최적화)도 Params로 외부화. 확인: w 기본값(예: 8:1 상당), 저신호 임계 정의(절대 신호 vs 국소 SNR), 감쇠 곡선 형태.
4. **[확인] SKS 추정 세부 — ×8 다운샘플 pyramid 재사용·커널 컨볼루션·마스크 화소 처리** — SWR-1101 다운샘플(×8)·이중 Gaussian 컨볼루션·마스크 처리 상세. **가정 default**: (a) ×8 다운샘플은 `common/pyramid.py` Gaussian `reduce_once`(×2 3회)를 소비(SWR-000-9 ① pyramid 소비자, 중복 다운샘플 회피), 업샘플은 SWR-1102 명시 bilinear(scipy, pyramid Gaussian expand와 구별). **공간 도메인이므로 `common/fft_psd.py`가 아니라 `common/pyramid.py`를 소비**(T7 grid의 주파수 도메인 fft_psd와 구별).
