C# 검증 뷰어 재설계 — 스테이지 복수 선택·조합 검증

문서 위치. 이 HTML은 화면 방향을 설명하는 비규범 시각 자료다. 전체 algorithm catalog·9-family 계약·job/cancel·availability/evidence·artifact/run-manifest·인수 기준은 SPEC-XGUI-MASTER v0.5.1과 SPEC-XSEAM-002 v0.5.1이 우선한다. 사용자 승인·기준선 동결 전 구현은 허가되지 않는다.

재설계는 목적(알고리즘 그룹)별 탭 구조다 — 탭을 없애는 게 아니라 "시임 함수"로 갈린 현행 탭을 WP 그룹으로 재편한다. 탭마다 입력세트·파라미터·뷰어가 다르고, 특히 Calibration 탭은 OC·GC·BPM을 세트로 생성 → 촬영영상에 체크박스로 적용하는 워크플로다. 뷰어는 풀해상도 회색조(512² 의사색 폐기).

핵심 발견. run_pipeline(PipelineDefinition(stages=…), registry, calib_map, params_map, validation_mode) 하나로 — (1) 스테이지별 개별 calib 입력(calib_map), (2) 임의 정렬 부분집합(CANONICAL_ORDER의 부분수열만), (3) validation_mode→intermediates[]로 단일 실행에서 모든 스테이지 전/후 스크럽 — 이 조합 능력이 골든에 이미 존재합니다. 알고리즘 변경 0. 갭은 시임(IXdetEngine이 run_pipeline 미러 없음)과 문서(SPEC-XSEAM-001이 조합을 명시적 제외)에만 있습니다.
진단 (왜 "히스토그램"인가). ScottPlot이 512² 블록평균을 Viridis 의사색 heatmap으로 렌더 → 회색조 방사선영상이 아님. 36배 다운샘플로 검증 대상(bad pixel·line noise·defect 보간)이 화면 도달 전 소실. 탭이 "사용자 과제"가 아니라 "시임 함수"로 갈려 혼란.
C# WPF UI (DSP 0 · 요청 빌더) 언어중립 시임 (run_pipeline 미러 신설) FROZEN 알고리즘 (골든, 무변경) 등록 실측 가용 (offset/gain/defect)

0열기 → 보기 → 적용 → 저장 (사용자 워크플로)

사용자가 매일 하는 기본 동작이 먼저다 — 폴더를 지정해 그 안의 영상을 훑어보고, 결과로 어떤 파일이 어디에 나오는가. 열기는 모달 대화상자가 아니라 상주 폴더 브라우저(폴더 트리 + 썸네일 목록 + 형제 필름스트립 + 이전/다음)여야 한다 — IrfanView·XnView MP·Windows Photos·DICOM 뷰어 공통 구조.

현행 as-built 문제 (사용자가 실제로 부딪힘). ① 열기가 파일 하나 고르면 닫히는 모달 대화상자뿐 — 폴더를 지정해 그 안의 영상들을 훑어보는 브라우저가 없고, 등록 실측 영상은 3항목 알고리즘 드롭다운(offset/gain/defect)에 숨어 GHOST·nps는 접근 불가. ② 칼리브 결과가 입력과 동일한 <name>_result.raw(+.json)로 안 나옴 — 게다가 골든 io에 raw 쓰기 함수가 없음(load만, save 없음). 현재는 .npz+.json뿐, 볼 수 있는 영상도 안 나옴. ③ 4개 등록 흐름 중 2개(RegisteredPipeline·RealImageSanity)는 결과를 안 붙잡아 "실행 후 저장"이 조용히 실패. (C-20 가드는 정상)

1b탭 내부 적용 메커니즘 (Correction Stack)

중심 컨트롤. 스테이지는 엔진 고정순서로 잠기고(재정렬 불가, SWR-000-2), 사용자는 멤버십만 복수 선택. 각 행은 자기 calib 입력에 바인딩되며 등록 부재 시 비활성. Solo=개별 검증, 부분/전체 실행=조합, validation_mode 1회 실행으로 모든 전/후 스크럽.

골든 조합 계약 (딥싱크). 어댑터가 시임 뒤에서 호출할 실제 형상 — 조합 로직 재구현 없이 위임. 
# (a) 개별 스테이지
run_pipeline(frame, PipelineDefinition(stages=("offset",)), {"offset": offset.process},
             calib_map={"offset": off_cal}, params_map={...}, validation_mode=True)
# (b) 부분 조합 — 반드시 CANONICAL_ORDER의 부분수열 (재정렬 시 PipelineOrderError)
PipelineDefinition(stages=("offset","gain","defect"))
# (c) 전체 — post(미등록 tail) 제외한 구현 12 스테이지
stages = tuple(s for s in CANONICAL_ORDER if s != "post")
# 1-pass 전/후:  result.intermediates[i] = i번째 실행 스테이지 출력  → UI 스크럽
# 주의: registry는 stage→module.process (default_registry()의 ProcessModule 아님)

2UI 워크스페이스 — 좌측이 Correction Stack

탭 더미 제거. 좌=Correction Stack(조합 컨트롤), 중앙=단일 풀해상도 회색조 캔버스, 우=인스펙터, 하단=probe. 비교는 창이 아니라 모드.

3데이터 플로우 — run_pipeline · intermediates · W/L LUT

등록 영상은 풀해상도 1회 마샬. 조합 요청은 정렬 스테이지셋+calib_map을 골든에 위임, intermediates[]로 전/후 회수. W/L은 표시 전용 LUT.

4컴포넌트 아키텍처 — 시임에 run_pipeline 미러 신설

시임 계약·fidelity 테스트·골든은 유지. MainWindow·렌더 재구축. 시임은 run_pipeline 미러 + PipelineDefinition/calib_map DTO로 확장.
